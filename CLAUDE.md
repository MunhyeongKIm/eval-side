# Side Project Evaluator - Claude Code Team Mode

## 프로젝트 개요
사용자가 사이드 프로젝트 개념(텍스트 설명) 또는 진행중인 GitHub 링크를 제출하면,
여러 전문 에이전트가 다양한 관점에서 분석하고 종합 평가를 제공하는 시스템.

## 팀 구성 (Team Mode Agents)

평가는 다음 6개 전문 에이전트가 병렬로 수행:

| Agent | 역할 | 분석 영역 |
|-------|------|----------|
| `tech-analyst` | 기술 분석가 | 기술 스택, 아키텍처, 코드 품질, 확장성 |
| `market-analyst` | 시장 분석가 | 시장 규모, 경쟁 현황, 차별화 포인트, PMF |
| `ux-reviewer` | UX 리뷰어 | 사용자 경험, UI 디자인, 접근성, 온보딩 |
| `feasibility-checker` | 실현성 검토 | 개발 난이도, 리소스 요구사항, 일정 추정 |
| `growth-strategist` | 성장 전략가 | 수익 모델, 마케팅 채널, 성장 전략 |
| `risk-assessor` | 리스크 평가 | 기술적/법적/시장 리스크, 의존성 위험 |

## 워크플로우

1. **입력 수집**: 프로젝트 설명 텍스트 또는 GitHub URL
2. **정보 수집**: GitHub URL인 경우 repo 구조, README, 코드 분석
3. **병렬 분석**: 6개 에이전트가 동시에 각 영역 분석
4. **종합 평가**: 개별 분석을 종합하여 최종 리포트 생성
5. **출력**: 점수(100점 만점) + 상세 피드백 + 개선 제안

## 실행 방법

### 텍스트 개념 평가
```
/eval "나의 프로젝트 설명..."
```

### GitHub 링크 평가
```
/eval https://github.com/user/repo
```

### 팀 모드 직접 실행
```
/team 6:executor "프로젝트 평가: [입력]"
```

## 평가 기준 (Scoring Rubric)

각 영역 최대 점수:
- 기술 완성도: 20점
- 시장 적합성: 20점
- 사용자 경험: 15점
- 실현 가능성: 15점
- 성장 잠재력: 15점
- 리스크 관리: 15점

**총점: 100점**

## 출력 형식

최종 리포트는 `reports/` 디렉토리에 마크다운으로 저장됨.
파일명: `{project-name}-{date}.md`

## 개발 규칙
- 에이전트 프롬프트는 `agents/` 디렉토리에 관리
- 평가 템플릿은 `templates/` 디렉토리에 관리
- 과거 평가 리포트는 `reports/` 디렉토리에 보관
- UI 및 사용자 대면 텍스트는 영어로 작성
- 에이전트 평가 출력(분석, 리포트)은 한국어로 작성
